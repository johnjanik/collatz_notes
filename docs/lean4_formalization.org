* Preface: What Are We Formalizing?

Our proof strategy (see =next_steps.md= Items 9, 13--14, corrected 2026-02-16)
rests on four pillars:

1. *Topological rigidity.* The branch locus on $(\mathbb{Z}/k\mathbb{Z})^2$
   grows with sampling depth: 9,415 branch cells at $N = 10^8$, 16,371 at
   $N = 10^{10}$ (k=144).  The branch count has /not/ converged, so the genus
   is not yet defined.  However, the tunnel structure (branch + pure-even)
   is stable and the core proof uses the reflecting boundary, not the genus.

2. *SFT constraint.* The parity sequence is a golden mean shift (forbidden
   bigram "11").  This alone restricts $p_{\rm odd} \le 1/\varphi^2 = 0.382$,
   within 0.005 of the equilibrium threshold 0.387.

3. *Reflecting boundary.* Pure-even tunnel walls flank the branch strip on the
   Diophantine torus.  The irrational foliation passes /through/ the strip
   ($A \cap L \neq \emptyset$), but the walls act as a restoring force: any
   trajectory approaching the tunnel edge is forced into even steps, pushing
   $u(t)$ down.

4. *Baker persistence.* Baker's theorem on linear forms in logarithms
   guarantees the tunnel walls have minimum width $\ge C/k^{1+\epsilon}$ at
   every scale $k = 3^b$---the reflecting boundary is permanent.

A Lean 4 proof must encode each pillar as a formal theorem, then compose them
into the conclusion: every trajectory reaches 1.

** The Proof Skeleton (informal)

#+begin_quote
*Theorem (Collatz Conjecture).* For every $n \ge 1$, the Collatz sequence
$n, T(n), T^2(n), \ldots$ eventually reaches 1.
#+end_quote

*Proof outline:*
1. Define $T : \mathbb{N}^+ \to \mathbb{N}^+$ and the walk
   $u(t) = \nu_2(t) - \log_2 3 \cdot \nu_3(t)$.
2. Show that $T(n)$ is always even after an odd step (the "11" constraint),
   so the parity SFT is the golden mean shift.
3. Show that $p_{\rm odd} \le 1/\varphi^2 < \log_2(3/2)^{-1} \cdot (\log_2 3 - 1)$
   ... /but this is insufficient/: $1/\varphi^2 = 0.382 < 0.387$, gap only 0.005.
   Individual trajectories can temporarily exceed the threshold.
4. Define the "tunnel" on $(\mathbb{Z}/k\mathbb{Z})^2$ for $k = 3^b$ and show
   the pure-even walls create a reflecting boundary.
5. Use Baker's theorem to show the tunnel wall width $W(b) > 0$ for all $b$.
6. Conclude: the walk $u(t)$ has negative mean drift /and/ is confined by
   reflecting walls, so it diverges to $+\infty$, meaning $T^t(n) \to 1$.
   $\square$

Step 3 is the "golden mean near-miss"---almost but not quite enough.
Steps 4--5 close the gap.  Step 6 is a probabilistic/ergodic argument that
must be made rigorous (the hardest part).

** Assessment

Baker's theorem is the long pole.  Everything else can be built incrementally.
A realistic strategy is to formalize Pillars 1--3 first (doable with ~6 months
of Mathlib development), then tackle Pillar 4 (Baker) as a dedicated project.

/Alternative/: if we can strengthen Pillar 3 to show that the reflecting
boundary argument works /without/ Baker (e.g., via a direct spectral gap
argument for the transfer operator on a finite graph), we can bypass
Baker entirely.  This is the "finite-state verification" path.

#+begin_src lean4
-- The Collatz function
def collatz : ℕ → ℕ
  | 0 => 0
  | n => if n % 2 = 0 then n / 2 else 3 * n + 1

-- The iterated Collatz sequence
def collatzSeq (n : ℕ) : ℕ → ℕ
  | 0 => n
  | k + 1 => collatz (collatzSeq n k)

-- The conjecture itself
def collatzConjecture : Prop :=
  ∀ n : ℕ, n ≥ 1 → ∃ k : ℕ, collatzSeq n k = 1
#+end_src

#+begin_quote
*Baker's Theorem.* If $\alpha_1, \ldots, \alpha_n$ are nonzero algebraic
numbers and $b_1, \ldots, b_n$ are integers, not all zero, then either
$\Lambda = b_1 \log \alpha_1 + \cdots + b_n \log \alpha_n = 0$ or
$|\Lambda| > H^{-C}$ where $H = \max |b_i|$ and $C$ depends only on
$n$ and the $\alpha_i$.
#+end_quote

For Collatz we need the case $n = 2$, $\alpha_1 = 2$, $\alpha_2 = 3$:

$|m \log 2 - n \log 3| > C / \max(|m|, |n|)^{1+\epsilon}$

Implemented in =CollatzLean/Baker.lean=.

Fully proved (zero =sorry=):

  independence + log injectivity)
- =linear_form_nonzero=: $m \log 2 + n \log 3 \neq 0$ for $(m,n) \neq (0,0)$

** Phase 4 (Alternative): Finite-State Bypass of Baker              :bridge:

If Baker's theorem is too expensive to formalize, there is an alternative
path that replaces the infinite-scale Baker argument with a finite computation.

*** TODO [#B] Transfer operator on the finite torus graph
:PROPERTIES:
:Effort:   2m
:END:

At $k = 144$, the torus has $144^2 = 20{,}736$ cells, of which 2,379 are
branch cells and the rest are empty or pure-even.  The Collatz dynamics
define a stochastic matrix $P$ on the non-empty cells.

If we can show that $P$ has a *spectral gap* (second eigenvalue $< 1$),
then the system is exponentially mixing and no trajectory can be trapped
in a non-terminating cycle.

*Advantage:* This is a /finite/ matrix computation---fully decidable,
no transcendental number theory needed.

*Disadvantage:* Requires =native_decide= or verified linear algebra
for a ~10,000 × 10,000 matrix.  Lean's kernel may struggle with this.

The finite-state path is faster but requires either a very efficient
verified linear algebra library or a custom oracle.

* Key Mathematical Gaps to Close (Before Coding)

These are *mathematical* questions that must be resolved /before/ we can
write the Lean proofs.  They are not Lean-specific---they are open problems
in our framework.

** OPEN Is the "11" constraint sufficient to bound p_odd /per trajectory/?

The golden mean shift gives $p_{\rm odd} \le 1/\varphi^2$ as a /topological/
bound (maximal measure).  But we need: for any individual Collatz trajectory
of length $T$, the fraction of odd steps satisfies $\nu_3/T \le 1/\varphi^2$.

Is this true?  Or can a finite trajectory temporarily exceed this bound?

If finite excursions are possible (likely), we need the stronger statement:
the /Cesaro average/ of $p_{\rm odd}$ converges to something $< p_{\rm eq}$.
This requires the ergodic theorem, which in turn requires the transfer
operator spectral gap.

** DONE Does the branch count converge as $N \to \infty$?

*Answer: No, it has not converged.*  The 10B run ($N = 10^{10}$) gives:

| k | N=10^8 | N=10^{10} | Change |
|-----+--------+-----------+--------|
| 144 | 9,415 | 16,371 | +74% |
| 729 | --- | 16,577 | --- |

The branch count is still growing.  The genus (Riemann-Hurwitz) is therefore
not yet defined as a finite quantity.  Pure-even walls also grew: 659 at k=144,
893 at k=729.

This does /not/ affect the core proof, which uses the tunnel reflecting
boundary (branch + pure-even), not the genus.  The tunnel structure remains
well-defined at every finite $N$.

** OPEN Can the spectral gap of the transfer operator on the $k = 144$ torus be computed?

This is the "finite-state bypass" question.  If we can construct the
transition matrix and verify its spectral gap, we avoid Baker entirely.
The matrix is ~10K × 10K---large but not impossible.

** DONE What is the precise relationship between the walk $u(t)$ diverging and $T^t(n) \to 1$?

*Resolved in Phase 4* (=Conclusion.lean=).  The bridge is:
1. The multiplicative identity: $T^t(n) \cdot 2^{\nu_2} = n \cdot 3^{\nu_3} + R(t)$.
2. A correction bound: $R(t) \le C \cdot 3^{\nu_3}$ (sorry'd as =correction_bound=).
3. Walk divergence gives $\nu_2 - \log_2(3) \cdot \nu_3 \to +\infty$, so
   $(n+C) \cdot 3^{\nu_3} < 2^{\nu_2}$ eventually.
4. Therefore $T^t(n) \cdot 2^{\nu_2} < 2 \cdot 2^{\nu_2}$, giving $T^t(n) = 1$.

The correction bound is the remaining hard sorry---simple induction fails
because the recurrence at odd steps introduces =collatzSeq= values.

* Appendix: Prior Art

- *lean4-collatz-proof-template* (GitHub: kaiarasanen): Basic Lean 4 template
  with inductive definition of reachability.
- *Affine recursion framework* (AMS Math. Comp., peer review): Uses
  mathematical induction and cycle elimination, verified in Lean 4.
- *Tao (2019)*: "Almost all Collatz orbits attain almost bounded values."
  Proven for density-1 set but not all $n$.  Tao's methods are probabilistic
  and do not directly translate to our deterministic framework.
- *Conway (1972)*: Collatz-type problems can be undecidable in general.
  Our approach sidesteps this by working with the /specific/ $(3n+1)/2^k$
  function, not a general class.
